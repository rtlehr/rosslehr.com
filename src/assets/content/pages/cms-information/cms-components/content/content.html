MainContentComponent Instructions

The MainContentComponent is designed to load an external HTML file from the assets folder and render its content directly into the browser window. It uses Angular's HttpClient to fetch the HTML file and sanitizes the content for safe rendering. After the content is loaded, the component observes the rendered DOM to attach click events, highlight code blocks, and detect modal triggers for enhanced interactivity.

Component Inputs

Input Name Type Description
fileToLoad String The path to the HTML file to load (e.g., 'content/main-content.html'). This file should be placed in the assets folder.
divId String An identifier for the main container that will display the loaded HTML content. Useful for styling or DOM manipulation.

HTML File Structure

Unlike components that load JSON data, the MainContentComponent expects an HTML file. The HTML file should contain valid markup and can include elements such as headings, paragraphs, code blocks, or custom modal triggers. For example: 


<h1>Welcome to Our Site</h1>
<p>This content is loaded from an external HTML file.</p>
<pre><code class="language-typescript">
  // Your code snippet here
</code></pre>
<trigger-modal data-content='{"title": "Modal Title", "body": "Modal Body Content"}'></trigger-modal>

The component processes the loaded HTML to attach dynamic behaviors such as code highlighting and modal triggers.
